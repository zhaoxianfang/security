## 🎯 分层架构详解

### 1. 客户端层 (Client Layer)

**职责**：用户交互界面

**组件**：
- Web Browser - 浏览器
- Mobile App - 移动应用
- API Client - API客户端
- Third Party - 第三方集成

**技术栈**：
- HTML/CSS/JavaScript
- React/Vue/Angular
- Mobile SDKs
- REST/GraphQL API

### 2. 网络层 (Network Layer)

**职责**：负载均衡和安全防护

**组件**：
- Load Balancer - 负载均衡器
- Reverse Proxy - 反向代理
- CDN - 内容分发网络
- Firewall - 防火墙

**技术栈**：
- Nginx
- Apache
- Cloudflare
- AWS ELB/ALB

### 3. 应用服务器层 (Server Layer)

**职责**：应用逻辑处理

**组件**：
- Laravel Application - Laravel应用
- PHP-FPM - PHP进程管理器
- Queue Workers - 队列工作进程

**技术栈**：
- Laravel 11+
- PHP 8.2+
- Redis/Database

### 4. 安全扩展包层 (Security Layer)

**职责**：安全检查和防护

**组件**：
- SecurityMiddleware - 安全中间件
- Handler Chain - 处理器链
- Services - 服务层
- Models - 数据模型
- Events - 事件系统

**设计模式**：
- 责任链模式
- 策略模式
- 工厂模式
- 装饰器模式
- 观察者模式

### 5. 数据层 (Data Layer)

**职责**：数据持久化和访问

**组件**：
- Database - 数据库
- Cache - 缓存
- Logs - 日志
- Storage - 存储

**技术栈**：
- MySQL/PostgreSQL/SQLite
- File Cache/Redis
- Log Files
- Local Storage/S3

### 请求处理流程

```
用户请求
    ↓
客户端
    ↓
负载均衡器
    ↓

    ↓

    ↓
Laravel应用
    ↓
SecurityMiddleware (安全中间件)
    ↓
┌─────────────────────────────────────┐
│  1. 资源路径检查 (ResourcePath)    │
│     ↓                              │
│  2. 白名单检查 (Whitelist)         │
│     ↓                              │
│  3. 内网IP检查 (IntranetIP)        │
│     ↓                              │
│  4. 限流检查 (RateLimit)          │
│     ↓                              │
│  5. 黑名单检查 (Blacklist)         │
│     ↓                              │
│  6. 威胁检测 (ThreatDetection)     │
│     ├─ SQL注入检测                │
│     ├─ XSS攻击检测                │
│     ├─ 命令注入检测                │
│     ├─ 路径遍历检测                │
│     └─ 文件上传检测                │
│     ↓                              │
│  7. 可疑User-Agent检查             │
│     ↓                              │
│  8. 可疑HTTP头检查                │
│     ↓                              │
│  9. URL路径检查                    │
│     ↓                              │
```

### 限流流程

```
请求到达
    ↓
RateLimiterService.checkRateLimit()
    ↓
┌─────────────────────────────────────┐
│  1. 获取客户端IP                   │
│     ↓                              │
│  2. 生成限流指纹                   │
│     ├─ IP地址                       │
│     ├─ 路径                         │
│     └─ HTTP方法                    │
│     ↓                              │
│  3. 检查内存缓存                    │
│     ├─ 命中 → 返回计数              │
│     └─ 未命中 → 继续检查           │
│     ↓                              │
│  4. 检查文件缓存                    │
│     ├─ 命中 → 更新内存缓存           │
│     └─ 未命中 → 检查数据库           │
│     ↓                              │
│  5. 检查限流窗口                    │
│     ├─ 秒级窗口 (60次/秒)          │
│     ├─ 分钟窗口 (1000次/分)       │
│     └─ 小时窗口 (10000次/时)       │
│     ↓                              │
│  6. 增加请求计数                    │
│     ├─ 原子操作                      │
│     └─ 文件锁保证                   │
│     ↓                              │
│  7. 检查是否超限                    │
│     ├─ 未超限 → 放行                │
│     └─ 超限 → 拦截                  │
│         ├─ 记录日志                  │
│         ├─ 封禁IP                    │
│         └─ 返回429错误              │
└─────────────────────────────────────┘
    ↓
返回检查结果
```

## ⏱️ 时序图

### 完整请求处理时序图

```
客户端 → 安全中间件: HTTP请求
安全中间件 → ResourcePathHandler: 处理请求
ResourcePathHandler → 安全中间件: 放行
安全中间件 → WhitelistHandler: 处理请求
WhitelistHandler → WhitelistService: 检查白名单
WhitelistService → 缓存: 查询缓存
缓存 → WhitelistService: 返回缓存
WhitelistService → 数据库: 查询数据库
数据库 → WhitelistService: 返回结果
WhitelistService → 安全中间件: 不在白名单
安全中间件 → IntranetIPHandler: 处理请求
IntranetIPHandler → IpHelper: 检查内网IP
IpHelper → IntranetIPHandler: 返回结果
IntranetIPHandler → 安全中间件: 不是内网IP
安全中间件 → RateLimitHandler: 处理请求
RateLimitHandler → RateLimiterService: 检查限流
RateLimiterService → 缓存: 查询限流计数
缓存 → RateLimiterService: 返回计数
RateLimiterService → 安全中间件: 未超限
安全中间件 → BlacklistHandler: 处理请求
BlacklistHandler → IpManagerService: 检查黑名单
IpManagerService → 缓存: 查询黑名单
缓存 → IpManagerService: 返回缓存
IpManagerService → 安全中间件: 不在黑名单
安全中间件 → ThreatDetectionHandler: 处理请求
ThreatDetectionHandler → ThreatDetectionService: 检测威胁
ThreatDetectionService → SqlInjectionDetector: SQL注入检测
SqlInjectionDetector → ThreatDetectionService: 无威胁
ThreatDetectionService → XssAttackDetector: XSS攻击检测
XssAttackDetector → ThreatDetectionService: 无威胁
ThreatDetectionService → 安全中间件: 无威胁
安全中间件 → IpManagerService: 记录访问
IpManagerService → 数据库: 插入记录
数据库 → IpManagerService: 插入成功
IpManagerService → 安全中间件: 记录完成
安全中间件 → 客户端: HTTP响应
```

### IP封禁时序图

```
安全中间件 → IpManagerService: banIp(request, 'SQLInjection', 75)
IpManagerService → 数据库: 查询IP
数据库 → IpManagerService: IP不存在
IpManagerService → ThreatDetectionService: 计算威胁评分
ThreatDetectionService → IpManagerService: 返回评分75
IpManagerService → IpManagerService: 计算封禁时长
IpManagerService → IpManagerService: 获取封禁原因
IpManagerService → 数据库: 添加到黑名单
数据库 → IpManagerService: 添加成功
IpManagerService → 事件总线: 触发IpCreated事件
事件总线 → LogIpCreated: 处理事件
LogIpCreated → 日志: 记录日志
日志 → LogIpCreated: 记录完成
事件总线 → SendIpAlert: 处理事件
SendIpAlert → 邮件服务: 发送告警邮件
邮件服务 → SendIpAlert: 邮件发送成功
IpManagerService → 缓存: 清除IP缓存
缓存 → IpManagerService: 清除完成
IpManagerService → 安全中间件: 封禁成功
```

## 🎯 总结

### 架构特点

1. **分层清晰**: 五层架构，职责明确
2. **松耦合**: 通过接口和依赖注入降低耦合度
3. **高可用**: 支持负载均衡、读写分离、缓存优化
4. **易扩展**: 插件化架构，支持动态扩展
5. **高性能**: 多层缓存、批量操作、懒加载优化
6. **高安全**: 四层安全防护，全面威胁检测

### 设计原则

1. **SOLID原则**: 单一职责、开闭、里氏替换、接口隔离、依赖倒置
2. **DRY原则**: 避免代码重复
3. **KISS原则**: 保持简单
4. **YAGNI原则**: 不要过度设计

### 设计模式

1. **责任链模式**: 中间件处理链
2. **策略模式**: 威胁检测策略
3. **工厂模式**: 对象创建
4. **装饰器模式**: 缓存装饰器
5. **观察者模式**: 事件监听
6. **模板方法模式**: 抽象基类
